# thin.css — Migration Guide: 2.0 → 3.0

## Overview

3.0 introduces a **musical scale system** for all sizing and spacing, two brand-new utility modules (`_app`, `_stabilize`), and a full **CSS Grid** module. Several class names were renamed for clarity, and spacing now uses CSS custom properties instead of compiled SCSS values.

---

## New Modules

| Module | Classes | Description |
|---|---|---|
| `_stabilize.scss` | `.stabilize` | Replaces the `%stabilize-window` Sass extend from 2.0. Apply to `html` or `body`. |
| `_app.scss` | `.app--viewport`, `.app--shell`, `.app--safePad` | New viewport/SPA shell/safe-area inset utilities. |
| `_gap.scss` | `.gap--*` | Gap utilities for flexbox and grid (not available in 2.0). |
| `_grid.scss` | `.grid`, `.grid--cols-*`, `.grid-item--*` | Full CSS Grid system (not available in 2.0). |

### Stabilize: 2.0 → 3.0

**2.0** — manually apply the Sass extend in your own SCSS:

```scss
html {
  @extend %stabilize-window;
  body { @extend %stabilize-window; }
}
```

**3.0** — use the class directly in HTML:

```html
<html class="stabilize">
  ...
</html>
```

### App utilities (new in 3.0)

```html
<!-- Full-viewport block that respects mobile browser UI chrome -->
<div class="app--viewport"></div>

<!-- Fixed, full-screen SPA shell that clips overflow -->
<div class="app--shell"></div>

<!-- Safe-area inset padding for notched devices -->
<div class="app--safePad"></div>
```

## Spacing System Overhaul

All spacing modules (`padding`, `margin`, `gap`) now use **CSS custom properties** generated by the musical scale system instead of compiled SCSS values.

### Token reference

| Token | 2.0 value | 3.0 value | Notes |
|---|---|---|---|
| `xxs` | — | `0.125rem` | New in 3.0 |
| `xs` | `0.25rem` | `0.25rem` | Same value |
| `sm` | `0.5rem` | `0.5rem` | Same value |
| `md` | `1rem` | `1rem` | Same value |
| `lg` | `2rem` | `2rem` | Same value |
| `xl` | `4rem` | `4rem` | Same value |
| `xxl` | — | `8rem` | New in 3.0 |
| `xxxl` | — | `16rem` | New in 3.0 |

> Existing named sizes (`xs` through `xl`) produce identical pixel values — no visual regression expected for those tokens.

### Configuring the scale at runtime

The default is the Classic Typographic scale at the `web` key. Override via `data-` attributes on `:root` — no SCSS recompile needed.

```html
<!-- Default: classic scale, web key — no attribute required -->
<html>

<!-- Golden ratio scale -->
<html data-scale="golden">

<!-- Large-screen key -->
<html data-key="large">

<!-- Combined -->
<html data-scale="perfect-fifth" data-key="large">
```

**Available scales:** `tritonic`, `tetratonic`, `classic` (default), `perfect-fifth`, `golden`, `major-third`

**Available keys:** `web` (default), `large`, `small`, `print`

## Renamed Classes

### Padding

| 2.0 | 3.0 |
|---|---|
| `padding--x-{size}` | `padding--left-right-{size}` |
| `padding--y-{size}` | `padding--top-bottom-{size}` |
| `padding--all-{size}` | `padding--all-{size}` *(unchanged)* |
| `padding--top-{size}` | `padding--top-{size}` *(unchanged)* |
| `padding--bottom-{size}` | `padding--bottom-{size}` *(unchanged)* |
| `padding--left-{size}` | `padding--left-{size}` *(unchanged)* |
| `padding--right-{size}` | `padding--right-{size}` *(unchanged)* |

### Margin

| 2.0 | 3.0 |
|---|---|
| `margin--y-{size}` | `margin--left-right-{size}` |
| `margin--x-{size}` | `margin--top-bottom-{size}` |
| `margin--topLeft-{size}` | `margin--top-left-{size}` |
| `margin--topRight-{size}` | `margin--top-right-{size}` |
| `margin--bottomLeft-{size}` | `margin--bottom-left-{size}` |
| `margin--bottomRight-{size}` | `margin--bottom-right-{size}` |
| `margin--auto-y` | `margin--left-right-auto` |
| `margin--auto-x` | `margin--top-bottom-auto` |
| `margin--zero-y` | `margin--left-right-zero` |
| `margin--zero-x` | `margin--top-bottom-zero` |

> **Note:** In 2.0, `margin--y` applied left/right and `margin--x` applied top/bottom — this naming was counterintuitive. 3.0 corrects it with explicit `left-right` / `top-bottom` names.

## Typography System

Type sizing now uses the musical system via CSS custom properties instead of values computed at build time. The class naming convention is unchanged, but sizes are now dynamic and themeable.

### What changed

- **2.0:** font sizes were baked into compiled CSS at build time (e.g. `font-size: 1.1487rem`)
- **3.0:** font sizes reference CSS variables (e.g. `font-size: var(--musical-i)`) that respond to `data-scale` / `data-key` on `:root`

### Class names: no change required

`type--size-i` through `type--size-v` (and octave variants `+1`, `+2`, `-1`, etc.) work exactly as before. Weight and alignment utilities are also unchanged.

---

## `_reset.scss` Changes

3.0 adds font rendering to the global `html` element:

```css
/* New in 3.0 */
html {
  text-rendering: optimizeLegibility;
  -webkit-font-smoothing: antialiased;
  -moz-osx-font-smoothing: grayscale;
}
```

3.0 also adds the standard `appearance: none` (alongside `-webkit-appearance: none`) on `.reset` buttons and button-type inputs.

---

## `lib/_extends.scss` Changes

The `%stabilize-window` placeholder was **removed** from `lib/_extends.scss` in 3.0. Any custom SCSS that extends it must be updated to use the `.stabilize` class instead (see [Stabilize](#stabilize-20--30) above).
